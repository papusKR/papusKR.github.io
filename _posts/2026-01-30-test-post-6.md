# SIEM과 보안 모니터링

SIEM(Security Information and Event Management)은 실시간 보안 이벤트를 수집하고 분석하는 시스템입니다.

## 주요 SIEM 솔루션

### 1. Splunk
```spl
# 실패한 로그인 시도 검색
index=security sourcetype=auth 
| search action=failed 
| stats count by user, src_ip 
| where count > 5
```

### 2. ELK Stack
```json
{
  "query": {
    "bool": {
      "must": [
        { "match": { "event.category": "authentication" }},
        { "match": { "event.outcome": "failure" }}
      ],
      "filter": {
        "range": {
          "@timestamp": {
            "gte": "now-1h"
          }
        }
      }
    }
  }
}
```

## 로그 수집

### Filebeat 설정
```yaml
filebeat.inputs:
- type: log
  enabled: true
  paths:
    - /var/log/auth.log
    - /var/log/syslog

output.elasticsearch:
  hosts: ["localhost:9200"]
  index: "security-logs-%{+yyyy.MM.dd}"
```

## 이벤트 상관 분석

```python
# 이상 징후 탐지 예제
import pandas as pd
from sklearn.ensemble import IsolationForest

# 로그 데이터 로드
logs = pd.read_csv('security_logs.csv')

# 이상 징후 탐지 모델
model = IsolationForest(contamination=0.1)
anomalies = model.fit_predict(logs[['login_attempts', 'failed_auth']])
```

## 대시보드 구성

주요 메트릭:
- 실패한 인증 시도
- 비정상적인 네트워크 활동
- 권한 상승 이벤트
- 데이터 유출 의심 활동

## 알림 규칙

```yaml
alert:
  - name: Multiple Failed Logins
    query: failed_auth > 10
    timeframe: 5m
    action: email, slack
    
  - name: Privileged Account Activity
    query: user=root AND hour>22
    action: pagerduty
```

## 결론

효과적인 SIEM 운영을 위해서는 적절한 로그 수집, 상관 분석, 신속한 대응이 필요합니다.
