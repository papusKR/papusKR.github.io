# 클라우드 네트워크 아키텍처

클라우드 환경에서의 네트워크 설계는 온프레미스와 다른 접근 방식이 필요합니다.

## AWS VPC 설계

### 서브넷 구성
```bash
# Public Subnet
aws ec2 create-subnet \
  --vpc-id vpc-12345678 \
  --cidr-block 10.0.1.0/24 \
  --availability-zone ap-northeast-2a

# Private Subnet
aws ec2 create-subnet \
  --vpc-id vpc-12345678 \
  --cidr-block 10.0.2.0/24 \
  --availability-zone ap-northeast-2a
```

### 보안 그룹 설정
```bash
# Security Group 생성
aws ec2 create-security-group \
  --group-name web-sg \
  --description "Web Server Security Group" \
  --vpc-id vpc-12345678
```

## 네트워크 보안

### 1. 네트워크 ACL
- 서브넷 레벨의 방화벽
- Stateless 규칙
- 인바운드/아웃바운드 규칙 별도 설정

### 2. 보안 그룹
- 인스턴스 레벨의 방화벽
- Stateful 규칙
- 최소 권한 원칙 적용

### 3. VPC 피어링
```bash
# VPC Peering 연결 생성
aws ec2 create-vpc-peering-connection \
  --vpc-id vpc-11111111 \
  --peer-vpc-id vpc-22222222
```

## 로드 밸런싱

- **Application Load Balancer**: HTTP/HTTPS 트래픽
- **Network Load Balancer**: TCP/UDP 트래픽
- **Gateway Load Balancer**: 방화벽, IDS/IPS

## 모니터링

```bash
# CloudWatch 로그 활성화
aws logs create-log-group --log-group-name /aws/vpc/flowlogs

# VPC Flow Logs 설정
aws ec2 create-flow-logs \
  --resource-type VPC \
  --resource-ids vpc-12345678 \
  --traffic-type ALL
```

## 결론

클라우드 네트워크는 유연성과 보안을 동시에 고려해야 합니다.
